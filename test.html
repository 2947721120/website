cdnjs - local loader - beta

cdnjs local loader is an experimetal idea implemented with a chrome extension

cdnjs.com currently host a lot of javascript libraries and their entire version history.

We have simply downloaded the entire repository into a chrome extension. When your browser sends off a request for a javascript file, the extension quickly checks to see if the file is hosted on cdnjs.

If the file is hosted, instead of sending off a network request, we simply include the file from your chrome extension store.

So instead of waiting for an external server to respond, the file gets served locally as fast as your computer can physically.

Not only do we check for cdnjs.com request but we also check for Google CDN's because they share the same file structure as us

Try it

Reload this page and check out the network request speed without the extension. Make sure to try out a cold refresh to see first time downloads.

Now download the extension cdnjs-local-loader

Refresh the browser and see super fast page load speeds. All web pages using google and cdnjs CDNs will load faster and if you develop using cdn links, you can also have localhost development speeds still

Questions

How is this any different from by browser cache?

  • If the file isn't cached yet it, it will still load instantly.
  • There are still lookup times with caching so locally stored can get even faster.
  • If you are developing and refreshing your cache quite often then at least the third party libraries won't have to re-download.
  • You can run the plugin in offline mode and have an entire CDN's worth of libraries and their version history on the go.
  • Super simple to install chrome plugins and they auto update!
